# Contributing and review protocol

The most useful contribution is an independent exact verification or a finite countercalculation.

## Reporting verification

Use the **Independent verification report** issue form and include:

- the check or claim reproduced;
- operating system and exact software versions;
- the command or independent derivation;
- unedited output;
- whether the result matched exactly.

## Reporting a possible error

Use the **Mathematical issue** form. Identify one precise statement and provide the smallest reproducible objection. Broad statements that the Jacobian Conjecture is known to be open are not substitutes for checking the displayed polynomial identity and collision.

## Pull requests

Changes to Paper I or any frozen check require:

1. a clear explanation of the issue;
2. an exact regression test;
3. synchronized changes to the paper and check matrix;
4. explicit acknowledgment that the original frozen package remains available under `frozen/`.

Paper II changes must remain clearly labeled as held structural research and must not be presented as necessary for the central counterexample claim.
